## Histórico de Ações e Decisões

### 1. Correções Iniciais de Compilação
- **Erro de Codificação (C2018):** Removidos acentos/backticks perdidos em `SamplerPlayer/Source/PluginEditor.cpp`.
- **Mismatch de Assinatura:** Corrigido o tipo de retorno de `loadSoteroLibrary` em `PluginProcessor.cpp` (de `bool` para `void`) para coincidir com o header.
- **Cleanup:** Removido um `return true;` residual que impedia a compilação final do processador.

### 2. Refatoração para Correção de Linker (Unresolved Symbols)
- **Problema:** `PerformanceView` e `SetupView` estavam definidos no arquivo `.cpp` do Editor do Player, tornando-os inacessíveis para o Builder.
- **Solução:**
    - Criados `Common/SoteroViews.h` e `Common/SoteroViews.cpp`.
    - Extraídas as classes `PerformanceView`, `SetupView` e `LibraryDashboard` para estes novos arquivos.
    - Padronizado o namespace `sotero` para todos os componentes de UI compartilhados.
- **Estratégia de Build:** Configurado o módulo `Common` como uma `INTERFACE` library que compartilha fontes (`target_sources`). Isso permite que cada projeto (Player e Builder) compile os views com suas próprias configurações de ambiente, evitando conflitos de headers de sistema/SDK.

### 3. Integração de Interface (`ISoteroAudioEngine`)
- Implementada a interface em `SoteroBuilder/Source/MainComponent`.
- Desacoplada a UI dos processadores específicos, permitindo que os mesmos views funcionem tanto no Plugin (DAW) quanto no Builder (Standalone).
