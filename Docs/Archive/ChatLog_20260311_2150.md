## 2. Persistent Link Synchronization (Horizontal Drag)

## 4. Real-time Visual Synchronization
*   **Problem**: Resizing a sample in Layer A would move its "glued" neighbors, but the mirrored samples in Layer B would not move until the mouse was released, causing visual overlaps and lag.
*   **Fix**: Modified `updateColumnRegions` to explicitly call `updateFromMapping()` on all UI components in the column *before* recalculating their bounds. This forces Layer B to reflect the new collision-resolved positions in real-time.

## 5. The "Dancing Samples" Bug (Persistent Identity)
*   **Problem**: Moving a sample to another note and back changed its rank in the UI array. The sync system, which relied on array position (rank), would then link the wrong samples in Layer B.
*   **Fix**: Introduced a persistent `mappingIndex` in the `SampleRegion` class. lookup now uses this ID instead of array rank, making the mirror link indestructible.
